# F16: 댓글 기능 강화 - UX Flow

## 개요
기존 댓글 기능에 추천 Top3, 신고, 부적절 댓글 필터링을 추가한다.

---

## 사용자 여정

### Journey 1: 추천 Top3 댓글 확인

```
게시글 상세 → 댓글 섹션 진입
  → Top3 추천 댓글 섹션 노출 (좋아요 수 기준 상위 3개)
  → 일반 댓글 목록 아래에 표시
```

**트리거**: 댓글 섹션 로드 시 자동
**조건**: 좋아요 1개 이상인 댓글이 최소 1개 이상
**결과**: 최상단에 Top3 섹션, 아래에 전체 댓글 시간순 목록

### Journey 2: 댓글 신고

```
댓글 카드에서 더보기(⋮) 메뉴 탭
  → 팝업 메뉴에 "신고하기" 옵션 표시 (타인 댓글에만)
  → "신고하기" 탭 → 신고 사유 선택 Bottom Sheet 표시
  → 사유 선택 (스팸/광고, 욕설/비방, 불쾌한 내용, 저작권 침해, 기타)
  → "기타" 선택 시 상세 내용 입력 필드 표시
  → "신고하기" 버튼 탭 → 신고 접수 완료
  → SnackBar "신고가 접수되었습니다" 표시
```

**트리거**: 타인 댓글의 더보기 메뉴
**조건**: 로그인된 사용자, 본인 댓글이 아닌 경우
**결과**: Report 엔티티에 신고 기록 저장

### Journey 3: 부적절 댓글 필터링

```
댓글 목록 로드
  → 서버에서 isFiltered=true인 댓글 확인
  → 해당 댓글 내용을 "[부적절한 내용이 포함된 댓글입니다]"로 대체 표시
  → 필터링된 댓글은 좋아요/답글 버튼 비활성화
```

**트리거**: 댓글 목록 조회 시 자동 (백엔드에서 처리)
**조건**: 관리자가 부적절 처리한 댓글 또는 금칙어 감지 댓글
**결과**: 사용자에게 필터링된 댓글은 내용 대체되어 표시

## 화면 전환도

```
게시글 상세 (Post Detail)
  └── 댓글 섹션 (CommentSection) - 기존
       ├── [신규] Top3 추천 댓글 섹션
       │     └── CommentCard (하이라이트 스타일)
       ├── 전체 댓글 목록 - 기존
       │     └── CommentCard
       │           ├── 더보기 메뉴 (본인: 삭제 / 타인: 신고하기)
       │           └── [신규] 필터링 댓글 플레이스홀더
       └── [신규] 댓글 신고 Bottom Sheet
             ├── 사유 목록 (라디오 버튼)
             ├── 기타 상세 입력 (선택적)
             └── 신고하기 버튼
```
