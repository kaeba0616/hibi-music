## 7. AC 검증 체크리스트

### AC-F7-1: 사용자 프로필 조회
- [ ] 프로필 이미지 표시
- [ ] 닉네임, 아이디 표시
- [ ] 게시글/팔로워/팔로잉 수 표시
- [ ] 로딩/에러 상태 처리

### AC-F7-2: 사용자 팔로우
- [ ] 팔로우 버튼 표시 (타인 프로필)
- [ ] 탭 시 팔로우 상태로 변경
- [ ] 팔로워 수 +1 반영
- [ ] 낙관적 업데이트

### AC-F7-3: 사용자 언팔로우
- [ ] 팔로잉 버튼 탭 시 확인 다이얼로그
- [ ] 확인 시 팔로우 해제
- [ ] 팔로워 수 -1 반영

### AC-F7-4: 팔로워 목록 조회
- [ ] 팔로워 탭 탭 시 목록 화면 이동
- [ ] 팔로워 목록 표시
- [ ] 각 사용자에 팔로우 버튼 표시

### AC-F7-5: 팔로잉 목록 조회
- [ ] 팔로잉 탭 탭 시 목록 화면 이동
- [ ] 팔로잉 목록 표시
- [ ] 각 사용자에 팔로잉 버튼 표시

### AC-F7-6: 팔로우 피드
- [ ] 피드에 "전체" / "팔로잉" 탭 추가
- [ ] 팔로잉 탭 선택 시 팔로우한 사용자 게시글만 표시
- [ ] 팔로우 없을 시 Empty 상태 표시

### AC-F7-7: 사용자 게시글 목록
- [ ] 프로필 하단에 게시글 목록 표시
- [ ] 무한 스크롤 지원
- [ ] 게시글 없을 시 Empty 상태

## 8. 데이터 요구사항 (참고용)

> 이 섹션은 Step 3(DB 설계)를 위한 참고 정보입니다.

### 8.1 사용자 프로필 정보
- 사용자 ID
- 닉네임
- 아이디 (username)
- 프로필 이미지 URL
- 게시글 수
- 팔로워 수
- 팔로잉 수

### 8.2 팔로우 관계
- 팔로워 ID (팔로우하는 사용자)
- 팔로잉 ID (팔로우당하는 사용자)
- 팔로우 일시
