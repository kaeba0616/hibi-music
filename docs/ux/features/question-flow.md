# F10: Question (문의하기) 사용자 흐름

## 개요
FAQ에서 해결되지 않은 문의사항을 직접 작성하여 운영팀에 전달하고, 답변을 확인할 수 있는 기능

---

## Acceptance Criteria

### AC-F10-1: 문의하기 화면 진입
- **Given**: 로그인된 사용자
- **When**: FAQ 화면의 "문의하기" 버튼 또는 설정 > 문의하기 메뉴 클릭
- **Then**: 문의하기 작성 화면이 표시됨

### AC-F10-2: 문의 유형 선택
- **Given**: 문의하기 작성 화면
- **When**: 문의 유형 선택 영역 탭
- **Then**: 문의 유형 목록이 표시됨 (계정, 서비스 이용, 버그 신고, 기능 제안, 기타)

### AC-F10-3: 문의 작성
- **Given**: 문의하기 작성 화면
- **When**: 유형 선택, 제목, 내용 입력 후 제출 버튼 클릭
- **Then**: 문의가 등록되고 확인 메시지 표시

### AC-F10-4: 문의 내역 조회
- **Given**: 로그인된 사용자
- **When**: 설정 > 문의 내역 메뉴 진입
- **Then**: 본인이 작성한 문의 목록이 표시됨 (유형, 제목, 작성일, 답변 상태)

### AC-F10-5: 문의 상세 조회
- **Given**: 문의 내역 화면
- **When**: 문의 항목 클릭
- **Then**: 문의 상세 내용과 답변(있는 경우)이 표시됨

### AC-F10-6: 입력 유효성 검사
- **Given**: 문의하기 작성 화면
- **When**: 필수 항목(유형, 제목, 내용) 미입력 시 제출 시도
- **Then**: 에러 메시지와 해당 필드 강조 표시

### AC-F10-7: 비로그인 사용자 안내
- **Given**: 비로그인 사용자
- **When**: 문의하기 접근 시도
- **Then**: 로그인 필요 안내와 로그인 화면으로 이동 옵션 표시

## 문의 유형 정의

| 유형 | 설명 | 예시 |
|------|------|------|
| 계정 | 계정 관련 문제 | 로그인 불가, 비밀번호 재설정 문제 |
| 서비스 이용 | 기능 사용 관련 문의 | 특정 기능이 작동하지 않음 |
| 버그 신고 | 오류 및 버그 제보 | 앱 크래시, UI 깨짐 |
| 기능 제안 | 새로운 기능 요청 | "이런 기능이 있으면 좋겠어요" |
| 기타 | 위 카테고리에 해당하지 않는 문의 | 일반 문의, 제휴 문의 |

---

## 문의 상태 정의

| 상태 | 설명 | UI 표시 |
|------|------|---------|
| 접수됨 | 문의가 등록된 상태 | 회색 뱃지 "접수됨" |
| 처리중 | 운영팀에서 검토 중 | 파란색 뱃지 "처리중" |
| 답변완료 | 답변이 등록된 상태 | 초록색 뱃지 "답변완료" |

## AC ↔ 화면 매핑

| AC | 화면 | 검증 방법 |
|----|------|----------|
| AC-F10-1 | QU-01 | FAQ 또는 설정에서 문의 작성 화면 진입 |
| AC-F10-2 | QU-01 | 유형 선택 드롭다운/바텀시트 동작 |
| AC-F10-3 | QU-01, QU-02 | 제출 후 완료 화면 표시 |
| AC-F10-4 | QU-03 | 문의 목록 표시 및 필터/정렬 |
