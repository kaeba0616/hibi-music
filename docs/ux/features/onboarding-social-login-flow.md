# F13: 온보딩 & 소셜 로그인 - UX Flow

## 1. 개요
- Feature ID: F13
- Feature 이름: 온보딩 & 소셜 로그인
- 설명: 앱 시작 시 온보딩 화면을 제공하고, 카카오/구글/네이버 소셜 로그인을 지원한다.
- Phase: 4 (Enhancement & Notion PRD Sync)

---

## 2. Acceptance Criteria

### AC-F1-5: 온보딩
- Given: 앱 최초 진입 또는 로그아웃 상태
- When: 앱 시작
- Then: 온보딩 페이지 표시 (메인 이동, 로그인, 회원가입 버튼)

### AC-F1-6: 소셜 로그인
- Given: 온보딩 또는 로그인 화면
- When: 카카오/구글/네이버 소셜 로그인 버튼 클릭
- Then: 해당 소셜 인증 진행 후 로그인 완료, 메인 화면 이동

---

## 3. 사용자 여정

### 3.1 진입 조건
- 앱 최초 설치 후 실행
- 로그아웃 후 앱 재진입
- 토큰 만료 후 앱 재진입

### 3.2 정상 플로우

#### 플로우 A: 비로그인 → 둘러보기
```
온보딩 (OB-01)
  └─ "둘러보기" 버튼 탭
      └─ 메인 화면 (오늘의 곡) → 비로그인 상태로 진입
```

#### 플로우 B: 비로그인 → 이메일 로그인
```
온보딩 (OB-01)
  └─ "로그인" 버튼 탭
      └─ 로그인 화면 (OB-02)
          ├─ 이메일 + 비밀번호 입력 → "로그인" 버튼 탭
          │   └─ 인증 성공 → 메인 화면 이동
          └─ 인증 실패 → 오류 메시지 표시
```

#### 플로우 C: 비로그인 → 소셜 로그인
```
온보딩 (OB-01) 또는 로그인 화면 (OB-02)
  └─ 소셜 로그인 버튼 (카카오/구글/네이버) 탭
      └─ 소셜 인증 대기 화면 (OB-03)
          ├─ 인증 성공 (기존 회원) → 메인 화면 이동
          ├─ 인증 성공 (신규 회원) → 닉네임 설정 후 메인 화면 이동
          └─ 인증 실패/취소 → 이전 화면으로 복귀 + 오류 메시지
```

#### 플로우 D: 비로그인 → 회원가입
```
온보딩 (OB-01)
  └─ "회원가입" 버튼 탭
      └─ 기존 회원가입 플로우 (이메일 등록 페이지로 이동)
```

### 3.3 대체 플로우

| 상황 | 처리 |
|------|------|
| 이미 로그인 상태로 앱 시작 | 온보딩 건너뛰고 메인 화면 직접 이동 |
| 소셜 인증 중 사용자가 취소 | 이전 화면으로 복귀, 안내 메시지 표시 |
| 소셜 로그인 네트워크 오류 | 오류 메시지 표시, 재시도 안내 |
| 소셜 계정이 이미 다른 방식으로 가입됨 | "이미 가입된 계정입니다" 안내 |

## 4. AC ↔ 화면 매핑

| AC ID | AC 설명 | 검증 화면 | 주요 UI 요소 |
|-------|---------|----------|-------------|
| AC-F1-5 | 온보딩 페이지 표시 | OB-01 온보딩 | 로고, 둘러보기/로그인/회원가입 버튼 |

| AC-F1-6 | 로그인 완료 → 메인 이동 | 메인 화면 | 자동 라우팅 |
