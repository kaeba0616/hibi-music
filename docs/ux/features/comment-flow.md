# Comment & Reaction (F6) - User Flow

## 1. Feature 개요

### 1.1 목표
게시글에 댓글을 작성하고 소통하는 기능

### 1.2 AC 목록
| AC ID | 설명 |
|-------|------|
| AC-F6-1 | 댓글 목록 조회 |
| AC-F6-2 | 댓글 작성 |
| AC-F6-3 | 댓글 삭제 |
| AC-F6-4 | 대댓글 작성 |
| AC-F6-5 | 댓글 좋아요 |

---

## 2. 사용자 여정

### 2.1 진입점
- **게시글 상세 화면** (PO-03)에서 댓글 섹션으로 스크롤
- **게시글 목록** (PO-01)에서 댓글 아이콘 탭

### 2.2 주요 시나리오

#### 시나리오 1: 댓글 조회 (AC-F6-1)
```
[게시글 상세 화면]
       ↓
[댓글 섹션 스크롤]
       ↓
[댓글 목록 확인]
```

#### 시나리오 2: 댓글 작성 (AC-F6-2)
```
[게시글 상세 화면]
       ↓
[댓글 입력창 탭]
       ↓
[키보드 활성화]
       ↓
[댓글 내용 입력]
       ↓
[전송 버튼 탭]
       ↓
[댓글 생성 완료]
       ↓
[목록에 새 댓글 표시]
```

#### 시나리오 3: 댓글 삭제 (AC-F6-3)
```
[본인 댓글]
     ↓
[더보기 메뉴 탭]
     ↓
[삭제 선택]
     ↓
[확인 다이얼로그]
     ↓
[삭제 확인]
     ↓
[댓글 삭제 완료]
```

#### 시나리오 4: 대댓글 작성 (AC-F6-4)
```
[댓글 목록]
     ↓
[답글 버튼 탭]
     ↓
[입력창에 @작성자 표시]
     ↓
[대댓글 내용 입력]
     ↓
[전송 버튼 탭]
     ↓
[대댓글 생성]
     ↓
[원 댓글 아래 표시]
```

#### 시나리오 5: 댓글 좋아요 (AC-F6-5)
```
[댓글]
  ↓
[좋아요 버튼 탭]
  ↓
[좋아요 토글]
  ↓
[UI 즉시 반영]
```

## 4. 상태 전이

### 4.1 댓글 작성 상태
```
[빈 입력창] → [텍스트 입력 중] → [전송 가능] → [전송 중] → [완료]
                                       ↓
                                  [전송 실패] → [재시도]
```

### 4.2 좋아요 상태
```
[좋아요 안함 (♡)] ←→ [좋아요 함 (♥)]
```

## 5. AC → 화면 매핑

| AC ID | 화면 | 검증 포인트 |
|-------|------|-------------|
| AC-F6-1 | PO-03 (댓글 섹션) | 댓글 목록이 작성자, 내용, 시간과 함께 표시됨 |
| AC-F6-2 | PO-03 (댓글 입력창) | 댓글 입력 후 전송 시 목록에 추가됨 |
| AC-F6-3 | PO-03 (댓글 더보기) | 삭제 확인 후 목록에서 제거됨 |
| AC-F6-4 | PO-03 (답글 버튼) | 대댓글이 원 댓글 아래 들여쓰기되어 표시됨 |
| AC-F6-5 | PO-03 (댓글 좋아요) | 좋아요 버튼 탭 시 상태 토글 및 카운트 변경 |

## 6. 에러 처리

### 6.1 댓글 작성 실패
- 네트워크 오류 시 "댓글 작성에 실패했습니다" 토스트
- 입력창 내용 유지

### 6.2 댓글 삭제 실패
- 네트워크 오류 시 "댓글 삭제에 실패했습니다" 토스트
- 댓글 유지

### 6.3 빈 댓글 작성 시도
- 전송 버튼 비활성화 (텍스트 없으면)

---

## 7. 특이사항

### 7.1 대댓글 구조
- 최대 1단계 대댓글만 지원 (대대댓글 X)
- 대댓글은 원 댓글 아래 들여쓰기로 표시
- 대댓글 입력 시 @원작성자 자동 추가

### 7.2 삭제 정책
- 본인 댓글만 삭제 가능
- 대댓글이 있는 원 댓글 삭제 시: "삭제된 댓글입니다" 표시 (대댓글 유지)
- 대댓글 삭제 시: 완전 삭제

### 7.3 정렬
- 기본: 작성일 오름차순 (오래된 순)
- 대댓글: 원 댓글 바로 아래, 작성일 오름차순
