## 4. 공통 컴포넌트

### 4.1 ArtistCard
아티스트 목록에서 사용하는 카드 컴포넌트

```dart
ArtistCard(
  artist: Artist,
  onTap: () => navigateToDetail,
  isFollowing: bool,
)
```

### 4.2 FollowButton
팔로우 버튼 컴포넌트

```dart
FollowButton(
  isFollowing: bool,
  onTap: VoidCallback,
  followerCount: int?,
)
```

### 4.3 SongListTile
노래 목록 아이템 컴포넌트

```dart
SongListTile(
  song: Song,
  onTap: () => navigateToSongDetail,
)
```

## 5. AC 검증 체크리스트

### AC-F3-1: 아티스트 목록 조회
- [ ] 목록 화면(AR-01)에 아티스트 그리드가 표시됨
- [ ] 각 카드에 아티스트 이름이 표시됨
- [ ] 각 카드에 프로필 이미지가 표시됨
- [ ] 각 카드에 곡 수가 표시됨

### AC-F3-2: 아티스트 상세 조회
- [ ] 상세 화면(AR-02)에 아티스트 프로필이 표시됨
- [ ] 상세 화면에 아티스트 소개가 표시됨
- [ ] 상세 화면에 노래 목록이 표시됨

### AC-F3-3: 아티스트 팔로우
- [ ] 상세 화면(AR-02)에 팔로우 버튼이 표시됨
- [ ] 팔로우 버튼 클릭 시 상태가 토글됨
- [ ] 팔로우 상태가 즉시 UI에 반영됨

### AC-F3-4: 팔로우 중인 아티스트 필터
- [ ] 목록 화면(AR-01)에 필터 탭이 표시됨
- [ ] "팔로우 중" 탭 선택 시 팔로우한 아티스트만 표시됨
- [ ] 필터 상태가 유지됨

## 6. 데이터 요구사항 (참고용)

> ⚠️ 이 섹션은 Step 3(DB 설계)를 위한 참고 정보입니다.

### 6.1 아티스트 정보
- 아티스트 ID
- 이름 (한글)
- 이름 (영문)
- 이름 (일본어)

- 노래 수

- 사용자 ID
- 아티스트 ID

- 팔로우 일시

### 6.3 노래 정보 (F2와 연동)
- 노래 ID
- 아티스트 ID
- 곡명
- 앨범 정보
- 발매일
