# Post (F5) - UX Flow

## 1. 개요

### 1.1 기능 설명
JPOP 관련 게시글을 작성하고, 노래를 태그하여 팬들과 소통하는 커뮤니티 기능

### 1.2 관련 AC
- AC-F5-1: 게시글 목록 조회
- AC-F5-2: 게시글 작성
- AC-F5-3: 게시글 상세 조회
- AC-F5-4: 노래 태그
- AC-F5-5: 게시글 수정
- AC-F5-6: 게시글 삭제
- AC-F5-7: 게시글 좋아요

---

## 2. 사용자 여정 (User Journey)

### 2.1 Entry Points
| 진입점 | 조건 | 목적지 |
|--------|------|--------|
| 하단 탭 "피드" 클릭 | 로그인 상태 | 게시글 목록 화면 |
| 푸시 알림 (게시글 좋아요/댓글) | 로그인 상태 | 게시글 상세 화면 |
| 사용자 프로필에서 게시글 탭 | 로그인 상태 | 해당 사용자의 게시글 목록 |

### 2.2 Exit Points
| 종료점 | 조건 | 다음 목적지 |
|--------|------|------------|
| 태그된 노래 클릭 | 노래 태그 존재 | 노래 상세 화면 (F2) |
| 태그된 아티스트 클릭 | 아티스트 태그 존재 | 아티스트 상세 화면 (F3) |
| 작성자 프로필 클릭 | 게시글 상세에서 | 사용자 프로필 화면 |
| 뒤로가기 | 어디서나 | 이전 화면 |

## 4. 화면 목록

| 화면 ID | 화면명 | Route | 설명 |
|---------|--------|-------|------|
| PO-01 | 게시글 목록 (피드) | `/feed` (탭) | 최신 게시글 목록 표시 |
| PO-02 | 게시글 작성 | `/posts/create` | 새 게시글 작성 |
| PO-03 | 게시글 상세 | `/posts/:postId` | 게시글 상세 및 댓글 |
| PO-04 | 게시글 수정 | `/posts/:postId/edit` | 게시글 수정 |
| PO-05 | 노래 선택 | (바텀시트) | 태그할 노래 검색/선택 |

## 5. 주요 인터랙션

### 5.1 게시글 작성
```
[FAB 버튼 탭]
    │
    ├─ 게시글 작성 화면 이동
    │
    ├─ 텍스트 입력 (필수)
    │
    ├─ 이미지 첨부 (선택)
    │   └─ 갤러리에서 선택 또는 카메라
    │
    ├─ 노래 태그 (선택)
    │   └─ 노래 검색 → 선택 → 카드 미리보기
    │
    └─ 등록 버튼
        ├─ 성공 → 목록으로 이동 + 스낵바 "게시글이 등록되었습니다"
        └─ 실패 → 에러 토스트
```

### 5.2 노래 태그
```
[노래 태그 버튼 탭]
    │
    ├─ 노래 검색 바텀시트 표시
    │
    ├─ 검색어 입력 (디바운스 300ms)
    │
    ├─ 검색 결과 목록 표시
    │   └─ 노래 제목, 아티스트, 앨범 이미지
    │
    └─ 노래 선택
        └─ 바텀시트 닫힘 + 게시글에 노래 카드 추가
```

### 5.3 게시글 좋아요
```
[좋아요 버튼 탭]
    │
    ├─ 낙관적 UI 업데이트 (즉시 반영)
    │
    ├─ API 호출
    │
    └─ 실패 시 롤백
```

### 5.4 게시글 삭제
```
[삭제 버튼 탭]
    │
    ├─ 확인 다이얼로그 표시
    │   "게시글을 삭제하시겠습니까?"
    │
    ├─ 확인 클릭
    │   ├─ 성공 → 목록으로 이동 + 스낵바 "삭제되었습니다"
    │   └─ 실패 → 에러 토스트
    │
    └─ 취소 클릭 → 다이얼로그 닫힘
```

## 6. 에러 처리

| 에러 상황 | 사용자에게 보여줄 내용 | 액션 |
|-----------|----------------------|------|
| 네트워크 에러 | "연결을 확인해주세요" | 재시도 버튼 |
| 서버 에러 | "잠시 후 다시 시도해주세요" | 재시도 버튼 |
| 게시글 작성 실패 | 토스트 "게시글 등록에 실패했습니다" | - |
| 좋아요 실패 | 토스트 "좋아요 처리 실패" | 자동 롤백 |
| 삭제 실패 | 토스트 "삭제에 실패했습니다" | - |
| 수정 실패 | 토스트 "수정에 실패했습니다" | - |
| 이미지 업로드 실패 | 토스트 "이미지 업로드 실패" | 다시 선택 |

## 7. 특수 케이스

### 7.1 게시글 없음 (Empty)
- 피드에 게시글이 없는 경우
- 메시지: "아직 게시글이 없습니다"
- 서브 메시지: "첫 번째 게시글을 작성해보세요!"

### 7.2 본인 게시글
- 수정/삭제 버튼 표시
- 더보기 메뉴 (⋮)에 수정/삭제 옵션

### 7.3 타인 게시글
- 좋아요, 댓글만 가능
- 신고 옵션 (향후 F11)

### 7.4 노래 태그 제한
- 게시글당 최대 1곡 태그 가능 (MVP)
- 향후 여러 곡 태그 가능하도록 확장 예정
