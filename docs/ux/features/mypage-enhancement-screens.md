# F17: My Page Enhancement (마이페이지 강화) 화면 구조

## MP-01: 내가 쓴 댓글 목록

### 화면 구조
```
┌─────────────────────────────────┐
│ ← 내가 쓴 댓글                  │  AppBar
├─────────────────────────────────┤
│                                 │
│ ┌─────────────────────────────┐ │
│ │ "정말 좋은 곡이죠! 저도..."  │ │  댓글 내용 (최대 2줄, ellipsis)
│ │ 夜に駆ける - YOASOBI         │ │  곡 제목 - 아티스트
│ │ 2026.03.15                  │ │  작성일
│ └─────────────────────────────┘ │
│                                 │
│ ┌─────────────────────────────┐ │
│ │ "이 노래 덕분에 JPOP에..."  │ │
│ │ Lemon - 米津玄師             │ │
│ │ 2026.03.14                  │ │
│ └─────────────────────────────┘ │
│                                 │
│ ┌─────────────────────────────┐ │
│ │ "@음악러버 앨범 전체가..."   │ │
│ │ アイドル - YOASOBI            │ │
│ │ 2026.03.13                  │ │
│ └─────────────────────────────┘ │
│                                 │
│         (스크롤)                │
└─────────────────────────────────┘
```

### UI 요소
- **AppBar**: 뒤로가기 버튼 + "내가 쓴 댓글" 타이틀
- **댓글 카드** (ListTile 스타일):
  - 댓글 내용: 최대 2줄, 초과 시 `...` (ellipsis)
  - 곡 제목 + 아티스트: 서브타이틀 (작은 텍스트, 회색)
  - 작성일: trailing 위치 또는 하단 (yyyy.MM.dd)
- **빈 상태**: 댓글이 없을 때 "아직 작성한 댓글이 없어요" 메시지
- **탭 동작**: 해당 곡 상세 페이지(`/song/:songId`)로 이동

### 인터랙션
- 스크롤: 전체 댓글 목록 스크롤
- 탭: 댓글 항목 탭 시 해당 곡 상세 화면으로 이동

## MP-02: 설정 - 푸시 알림 토글

### 화면 구조 (기존 설정 화면에 추가)
```
┌─────────────────────────────────┐
│ ← setting                       │  기존 AppBar
├─────────────────────────────────┤
│                                 │
│ 🔔 푸시 알림          [토글 ON] │  SwitchListTile
│    오늘의 곡 알림을 받습니다     │  subtitle
│                                 │
│ ─────────────────────────────── │  Divider
│                                 │
│ 🚪 SignOut                      │  기존 설정 항목들
│    로그아웃                      │
│                                 │
│ 🗑 delete                       │
│    아이디 삭제                   │
│                                 │
└─────────────────────────────────┘
```

### UI 요소
- **SwitchListTile**:
  - leading: 알림 아이콘
  - title: "푸시 알림"
  - subtitle: on일 때 "오늘의 곡 알림을 받습니다", off일 때 "알림이 꺼져있습니다"
  - trailing: Switch (on/off)

### 인터랙션
- 토글 변경 시 서버에 pushEnabled 상태 전송
- 변경 실패 시 이전 상태로 롤백 + 에러 스낵바

## MP-03: 문의 작성 (강화)

### 변경 사항 (기존 QU-01에 추가)
```
┌─────────────────────────────────┐
│ ← 문의하기                      │
├─────────────────────────────────┤
│                                 │
│ [기존 문의 유형 선택]            │
│                                 │
│ [기존 제목 입력]                 │
│                                 │
│ 내용 *                          │
│ ┌─────────────────────────────┐ │
│ │                             │ │  최소 10자
│ │                             │ │
│ │                             │ │
│ └─────────────────────────────┘ │
│ ⚠ 내용은 최소 10자 이상         │  10자 미만 시 에러
│                                 │
│ 📋 오늘 1/3개 문의 작성          │  일일 카운터
│                                 │
│ [기존 안내 메시지]               │
│                                 │
├─────────────────────────────────┤
│ [          문의 제출하기        ]│  3개 초과 시 비활성화
└─────────────────────────────────┘
```

### 추가된 UI 요소
- **일일 카운터**: "오늘 N/3개 문의 작성" 텍스트 (content 입력 아래)
  - 3개 도달 시 빨간색으로 표시
  - 제출 버튼 비활성화
- **내용 최소 길이 검증**: 기존 validation에 10자 미만 에러 추가 (이미 존재, 강조)

### 인터랙션
- 내용 10자 미만 제출 시: 에러 메시지 "내용은 최소 10자 이상 입력해주세요"
- 오늘 3개 초과 시: 제출 버튼 비활성화 + "오늘의 문의 작성 한도(3개)를 초과했습니다" 메시지
- 백엔드에서도 동일 검증 (이중 체크)
