# F14: 이메일 인증 & 회원가입 강화 - UX Flow

## 1. 개요
- Feature ID: F14
- 설명: 회원가입 시 이메일 인증 2단계 추가, 비밀번호 규칙 강화, 닉네임 변경 제한, 로그아웃 개선
- Phase: 4
- 기존 화면 확장: 회원가입 플로우(EmailView, PasswordView, NicknameView)

---

## 2. Acceptance Criteria

### AC-F1-7: 이메일 인증 (회원가입)
- Given: 회원가입 이메일 입력 화면
- When: 이메일 입력 후 인증 요청
- Then: 이메일 중복 체크 후 인증번호 발송, 인증확인 페이지로 이동

### AC-F1-8: 인증번호 확인
- Given: 인증확인 페이지
- When: 인증번호 입력 후 확인
- Then: 인증 완료 시 개인정보 등록 페이지로 이동 (이메일 자동 삽입)

### AC-F1-9: 닉네임 변경 제한
- Given: 로그인된 사용자, 프로필 수정 화면
- When: 닉네임 변경 시도
- Then: 1달 1회 제한 적용, 중복 및 금칙어 불가

### AC-F1-10: 비밀번호 규칙
- Given: 회원가입 또는 비밀번호 변경 화면
- When: 비밀번호 입력
- Then: 최소 8글자, 영문+숫자+특수기호 모두 포함 필수

### AC-F1-11: 로그아웃
- Given: 로그인된 사용자
- When: 마이페이지에서 로그아웃 버튼 클릭
- Then: 토큰/세션 삭제, 로그아웃 완료 안내 후 온보딩 페이지로 이동

## 3. 사용자 여정

### 3.1 이메일 인증 회원가입 플로우 (AC-F1-7, AC-F1-8, AC-F1-10)
```
온보딩 → "회원가입" 탭
  └─ EV-01: 이메일 입력
      ├─ 이메일 입력 → "인증번호 받기" 탭
      │   ├─ 중복 체크 성공 → 인증번호 발송 → EV-02로 이동
      │   └─ 중복 → "이미 가입된 이메일입니다" 에러
      └─ EV-02: 인증번호 확인
          ├─ 6자리 인증번호 입력 → "확인" 탭
          │   ├─ 인증 성공 → EV-03으로 이동 (이메일 자동 삽입)
          │   └─ 인증 실패 → "인증번호가 올바르지 않습니다" 에러
          ├─ "인증번호 재발송" → 새 인증번호 발송
          └─ 타이머 만료 (3분) → "인증번호가 만료되었습니다" → 재발송 안내
  └─ EV-03: 비밀번호 설정 (기존 PasswordView 강화)
      ├─ 비밀번호 규칙 실시간 체크 (8자↑, 영문, 숫자, 특수)
      ├─ 비밀번호 확인 일치 체크
      └─ "다음" → EV-04로 이동
  └─ EV-04: 닉네임 설정 (기존 NicknameView)
      └─ 중복/금칙어 체크 → 회원가입 완료 → 메인 화면
```

### 3.2 닉네임 변경 플로우 (AC-F1-9)
```
마이페이지 → 프로필 수정
  └─ 닉네임 변경 시도
      ├─ 1달 이내 변경 이력 → "닉네임은 한 달에 한 번만 변경할 수 있습니다" (변경 불가)
      └─ 변경 가능 → 중복/금칙어 체크 → 변경 완료
```

### 3.3 로그아웃 플로우 (AC-F1-11)
```
마이페이지 → "로그아웃" 탭
  └─ 확인 다이얼로그 "로그아웃 하시겠습니까?"
      ├─ "확인" → 토큰 삭제 → "로그아웃 되었습니다" 스낵바 → 온보딩 이동
      └─ "취소" → 마이페이지 유지
```
