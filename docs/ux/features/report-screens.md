## 데이터 모델

### Report (신고)
```dart
class Report {
  final int id;
  final int reporterId;        // 신고자 ID
  final ReportTargetType targetType;  // 대상 유형
  final int targetId;          // 대상 ID
  final ReportReason reason;   // 신고 사유
  final String? description;   // 상세 내용 (기타 선택 시)
  final ReportStatus status;   // 처리 상태
  final DateTime createdAt;
}

enum ReportTargetType {
  post,      // 게시글
  comment,   // 댓글
  member,    // 사용자
}

enum ReportReason {
  spam,         // 스팸/광고
  abuse,        // 욕설/비방
  inappropriate, // 불쾌한 내용
  copyright,    // 저작권 침해
  other,        // 기타
}

enum ReportStatus {
  pending,     // 대기중
  reviewed,    // 검토됨
  resolved,    // 처리완료
  dismissed,   // 기각
}
```

## 인터랙션 상세

### 신고 플로우
1. 더보기 메뉴에서 "신고하기" 탭
2. 바텀시트(RP-01) 표시
3. 신고 사유 선택 (라디오 버튼)
4. "기타" 선택 시 상세 입력 필드 표시
5. "신고하기" 버튼 탭
6. API 호출 (로딩 상태)
7. 성공 시 완료 다이얼로그(RP-02) 표시
8. "확인" 탭 → 바텀시트 닫기 → 이전 화면 복귀

### 중복 신고 처리
1. 신고 API 호출
2. 서버에서 중복 확인
3. 중복 시 409 Conflict 응답
4. 중복 안내 다이얼로그 표시

## 접근성

- 스크린 리더: 신고 사유에 적절한 레이블
- 키보드 네비게이션: 방향키로 사유 선택, Enter로 제출
- 충분한 터치 영역: 라디오 버튼 최소 높이 48dp
- 색상 대비: 선택된 항목 명확히 구분
