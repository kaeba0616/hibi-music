### 2.3 UI 요소 상세

#### 2.3.1 AppBar
| 요소 | 타입 | 동작 |
|------|------|------|
| 타이틀 | Text | "캘린더" |
| 필터 버튼 | IconButton | 좋아요 필터 토글 (AC-F4-4) |

#### 2.3.2 월 네비게이션 (AC-F4-3)
| 요소 | 타입 | 동작 |
|------|------|------|
| 이전 월 버튼 | IconButton (◀) | 이전 월로 이동 |
| 월 표시 | Text | "YYYY년 M월" 형식 |
| 다음 월 버튼 | IconButton (▶) | 다음 월로 이동 |
| 오늘 버튼 | TextButton | 현재 월, 오늘 날짜로 이동 |

#### 2.3.3 캘린더 그리드 (AC-F4-1)
| 요소 | 타입 | 동작 |
|------|------|------|
| 요일 헤더 | Row | 일~토 표시 |

| 마커 (•) | Container | 추천곡 있는 날짜 표시 |
| 선택 표시 (○) | CircleDecoration | 현재 선택된 날짜 |

| 요소 | 타입 | 동작 |

| 일본어 제목 | Text | 노래 일본어 제목 |

## 4. AC 검증 체크리스트

### AC-F4-1: 월별 캘린더 표시
- [ ] 캘린더 화면(CA-01)에 현재 월의 캘린더가 표시됨
- [ ] 요일 헤더(일~토)가 표시됨
- [ ] 날짜가 올바른 위치에 표시됨
- [ ] 추천곡이 있는 날짜에 마커(•)가 표시됨

### AC-F4-2: 날짜 선택
- [ ] 날짜 클릭 시 해당 날짜가 선택됨 (시각적 피드백)
- [ ] 선택된 날짜의 추천곡이 하단에 카드로 표시됨
- [ ] 추천곡이 없는 날짜 선택 시 Empty 메시지 표시

### AC-F4-3: 월 변경
- [ ] 이전 월 버튼(◀) 클릭 시 이전 월로 이동
- [ ] 다음 월 버튼(▶) 클릭 시 다음 월로 이동
- [ ] 월 표시가 "YYYY년 M월" 형식으로 업데이트됨
- [ ] 해당 월의 마커가 올바르게 표시됨

### AC-F4-4: 좋아요 필터
- [ ] 필터 버튼이 AppBar에 표시됨
- [ ] 필터 활성화 시 좋아요한 날짜만 마커 표시
- [ ] 필터 상태가 시각적으로 구분됨 (♡ ↔ ♥)
- [ ] 필터 적용 후 날짜 선택 시 해당 날짜 노래 표시

## 5. 데이터 요구사항 (참고용)

> 이 섹션은 Step 3(DB 설계) 및 Step 4(API 구현)를 위한 참고 정보입니다.

### 5.1 월별 노래 조회
- 입력: 년도, 월
- 출력: 해당 월의 추천곡 목록 (날짜, 노래 정보, 좋아요 여부)

### 5.2 날짜별 노래 조회
- 입력: 날짜 (YYYY-MM-DD)
- 출력: 해당 날짜의 추천곡 상세 정보

### 5.3 필터링 (클라이언트 사이드)
- 월별 조회 응답에 좋아요 여부가 포함되어야 함
- 클라이언트에서 좋아요 필터링 처리

## 6. 디자인 고려사항

### 6.1 마커 스타일
- 추천곡 있음: 작은 점 (Primary Color)
- 좋아요 있음: 작은 하트 또는 다른 색상 점 (선택적)

### 6.2 선택된 날짜 스타일
- 원형 배경색 (Primary Color with opacity)
- 텍스트 색상 변경

### 6.3 오늘 날짜 스타일
- 테두리 강조 (Primary Color border)
- 또는 배경색으로 구분

### 6.4 노래 카드 스타일
- F2 Daily Song의 카드 스타일 재사용
- 그림자, 라운드 코너 적용
- 앨범 이미지 비율 유지 (1:1 또는 4:3)
