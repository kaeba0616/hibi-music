# F14: 이메일 인증 & 회원가입 강화 - Screen Specifications

## 화면 목록
1. EV-01: 이메일 입력 (기존 EmailView 재구현)
2. EV-02: 인증번호 확인 (신규)
3. EV-03: 비밀번호 설정 (기존 PasswordView 강화)
4. 프로필 수정 - 닉네임 변경 제한 (기존 확장)

---

## 1. EV-01: 이메일 입력 (재구현)

### 기본 정보
- Route: 기존 Navigator.push 유지
- 기존 파일: `authentication/views/email_view.dart`
- 변경: 중복 체크 후 인증번호 발송 프로세스 추가

### UI 레이아웃
```
┌─────────────────────────────────────┐
│  ◀ 뒤로              HIBI          │
├─────────────────────────────────────┤
│                                     │
│  이메일을 입력해주세요              │  ← 안내 텍스트
│  인증번호를 보내드립니다            │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ 이메일                    ✕   │ │  ← TextField
│  │ 올바른 이메일 형식이 아닙니다  │ │  ← 에러 텍스트
│  └───────────────────────────────┘ │
│                                     │
│  ┌───────────────────────────────┐ │
│  │       인증번호 받기            │ │  ← Primary 버튼
│  └───────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘
```

### 상태별 UI
| 상태 | UI |
|------|-----|
| 초기 | 빈 이메일 필드, 버튼 비활성화 |
| 이메일 형식 오류 | 빨간 에러 텍스트 |
| 중복 이메일 | "이미 가입된 이메일입니다" 에러 |
| 발송 중 | 버튼에 로딩 인디케이터 |
| 발송 성공 | EV-02로 자동 이동 |

## 2. EV-02: 인증번호 확인 (신규)

### 기본 정보
- Route: Navigator.push (EV-01에서 이동)
- 신규 파일

### UI 레이아웃
```
┌─────────────────────────────────────┐
│  ◀ 뒤로              HIBI          │
├─────────────────────────────────────┤
│                                     │
│  인증번호를 입력해주세요            │
│  user@email.com 으로               │  ← 이메일 표시
│  인증번호를 보냈습니다              │
│                                     │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐  │
│  │  │ │  │ │  │ │  │ │  │ │  │  │  ← 6자리 입력
│  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘  │
│                                     │
│           02:47                     │  ← 카운트다운 타이머 (3분)
│                                     │
│  ┌───────────────────────────────┐ │
│  │          확인                  │ │  ← Primary 버튼
│  └───────────────────────────────┘ │
│                                     │
│     인증번호가 오지 않았나요?       │
│        [인증번호 재발송]            │  ← 텍스트 버튼
│                                     │
└─────────────────────────────────────┘
```

### UI 요소 상세
| 요소 | 타입 | 설명 |
|------|------|------|
| 이메일 표시 | Text | 입력한 이메일 주소 (볼드) |
| 인증번호 입력 | 6개 TextField | 각 1자리 숫자, 자동 포커스 이동 |
| 타이머 | Text | 3분 카운트다운, 만료 시 빨간색 |
| 확인 버튼 | ElevatedButton | 6자리 입력 완료 시 활성화 |
| 재발송 링크 | TextButton | 새 인증번호 발송 + 타이머 리셋 |

### 상태별 UI
| 상태 | UI |
|------|-----|
| 초기 | 빈 6자리 필드, 타이머 03:00, 확인 비활성화 |
| 입력 중 | 자동 포커스 이동, 6자리 완료 시 확인 활성화 |
| 인증 실패 | "인증번호가 올바르지 않습니다" 에러 + 필드 리셋 |
| 타이머 만료 | "인증번호가 만료되었습니다" 빨간 텍스트 + 재발송 강조 |
| 인증 성공 | EV-03으로 이동 |

## 3. EV-03: 비밀번호 설정 (강화)

### 기본 정보
- 기존 파일: `authentication/views/password_view.dart`
- 변경: 비밀번호 규칙 실시간 체크 UI 추가

### UI 레이아웃 (강화 부분)
```
┌─────────────────────────────────────┐
│  ◀ 뒤로              HIBI          │
├─────────────────────────────────────┤
│                                     │
│  비밀번호를 설정해주세요            │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ 비밀번호                  👁  │ │
│  └───────────────────────────────┘ │
│  ┌───────────────────────────────┐ │
│  │ 비밀번호 확인                 │ │
│  └───────────────────────────────┘ │
│                                     │
│  비밀번호 규칙:                     │  ← 실시간 체크
│  ✅ 8글자 이상                      │  ← 충족 시 초록
│  ❌ 영문 포함                       │  ← 미충족 시 빨간
│  ✅ 숫자 포함                       │
│  ❌ 특수기호 포함                   │
│  ✅ 비밀번호 일치                   │
│                                     │
│  ┌───────────────────────────────┐ │
│  │            다음                │ │
│  └───────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘
```

### 비밀번호 규칙 체크 항목
| 규칙 | 정규식 | 충족 시 | 미충족 시 |
|------|--------|--------|----------|
| 8글자 이상 | `.{8,}` | ✅ 초록 | ❌ 빨간 |
| 영문 포함 | `[a-zA-Z]` | ✅ 초록 | ❌ 빨간 |
| 숫자 포함 | `[0-9]` | ✅ 초록 | ❌ 빨간 |
| 특수기호 포함 | `[!@#$%^&*...]` | ✅ 초록 | ❌ 빨간 |
| 비밀번호 일치 | pw1 == pw2 | ✅ 초록 | ❌ 빨간 |

## 4. 프로필 수정 - 닉네임 변경 제한

### 변경 사항
기존 프로필 수정 화면의 닉네임 필드에 제한 로직 추가

### UI
```
닉네임 [현재닉네임      ]  [변경]

  ⚠ 닉네임은 한 달에 한 번만 변경할 수 있습니다
  마지막 변경일: 2026.03.01
  다음 변경 가능일: 2026.04.01
```

| 상태 | UI |
|------|-----|
| 변경 가능 | 닉네임 필드 활성화 + 변경 버튼 활성화 |
| 변경 불가 (1달 제한) | 필드 비활성화 + 다음 변경 가능일 표시 |
| 금칙어 | "사용할 수 없는 닉네임입니다" 에러 |
| 중복 | "이미 사용 중인 닉네임입니다" 에러 |
