# F13: 온보딩 & 소셜 로그인 - Screen Specifications

## 화면 목록
1. OB-01: 온보딩 화면
2. OB-02: 로그인 화면 (기존 확장)
3. OB-03: 소셜 인증 대기 화면

---

## 화면 상세

### 1. OB-01: 온보딩 화면

#### 기본 정보
- Route: `/onboarding`
- 타입: 랜딩 페이지
- 진입 조건: 비로그인 상태에서 앱 시작 시
- 기존 파일: `hibi_front/lib/features/onboarding/views/onboarding_view.dart` (재구현)

#### UI 레이아웃
```
┌─────────────────────────┐
│                         │
│                         │
│       [hibi 로고]        │
│     日々 - 매일 한 곡     │
│                         │
│                         │
│                         │
│  ┌───────────────────┐  │
│  │   둘러보기          │  │  ← 아웃라인 버튼 (메인 화면 이동)
│  └───────────────────┘  │
│                         │
│  ── 소셜 계정으로 시작 ──  │  ← 구분선 + 텍스트
│                         │
│  ┌─────┐┌─────┐┌─────┐ │
│  │카카오││구글 ││네이버│ │  ← 소셜 로그인 아이콘 버튼 (원형)
│  └─────┘└─────┘└─────┘ │
│                         │
│  ┌───────────────────┐  │
│  │   이메일로 로그인    │  │  ← Primary 버튼
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │   회원가입           │  │  ← 텍스트 버튼
│  └───────────────────┘  │
│                         │
└─────────────────────────┘
```

#### UI 요소 상세
| 요소 | 타입 | 설명 |
|------|------|------|
| 앱 로고 | Image | hibi 로고 이미지 (중앙 상단) |
| 앱 소개 텍스트 | Text | "日々 - 매일 한 곡" 서브타이틀 |
| "둘러보기" 버튼 | OutlinedButton | 비로그인 상태로 메인 화면 이동 |
| 소셜 구분선 | Divider + Text | "소셜 계정으로 시작" |
| 카카오 로그인 | IconButton (원형) | 카카오 노란색 (#FEE500), K 아이콘 |
| 구글 로그인 | IconButton (원형) | 구글 화이트 (#FFFFFF), G 아이콘 |
| 네이버 로그인 | IconButton (원형) | 네이버 그린 (#03C75A), N 아이콘 |
| "이메일로 로그인" 버튼 | ElevatedButton | Primary 색상, OB-02로 이동 |
| "회원가입" 버튼 | TextButton | 기존 회원가입 플로우(이메일 등록)로 이동 |

#### 상태별 UI
| 상태 | UI |
|------|-----|
| 기본 | 위 레이아웃대로 표시 |
| 소셜 버튼 탭 | 버튼 비활성화 + OB-03으로 전환 |

#### 인터랙션
- "둘러보기" 탭 → `context.go('/home')` (메인 화면, 비로그인)
- "이메일로 로그인" 탭 → `context.push('/login')` (OB-02)
- "회원가입" 탭 → `context.push('/signup')` (기존 회원가입 플로우)
- 카카오/구글/네이버 아이콘 탭 → OB-03 소셜 인증 대기 화면 표시

### 2. OB-02: 로그인 화면 (기존 확장)

#### 기본 정보
- Route: `/login`
- 타입: 폼
- 기존 파일: `hibi_front/lib/features/authentication/views/login_view.dart` + `login_form_view.dart`
- 변경 사항: 기존 로그인 화면에 소셜 로그인 버튼 추가, 두 화면 통합

#### UI 레이아웃
```
┌─────────────────────────┐
│ ← (뒤로가기)     HIBI    │  ← AppBar
│                         │
│       [hibi 로고]        │
│                         │
│  ┌───────────────────┐  │
│  │ 이메일              │  │  ← TextField (이메일)
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │ 비밀번호         👁  │  │  ← TextField (비밀번호, obscure)
│  └───────────────────┘  │
│                         │
│  ┌───────────────────┐  │
│  │     로그인          │  │  ← Primary 버튼
│  └───────────────────┘  │
│                         │
│  ── 또는 ──             │  ← 구분선
│                         │
│  ┌─────┐┌─────┐┌─────┐ │
│  │카카오││구글 ││네이버│ │  ← 소셜 로그인 아이콘 버튼
│  └─────┘└─────┘└─────┘ │
│                         │
│   계정이 없으신가요?      │
│   [회원가입]             │  ← 텍스트 링크
│                         │
└─────────────────────────┘
```

#### UI 요소 상세
| 요소 | 타입 | 설명 |
|------|------|------|
| AppBar | AppBar | 뒤로가기 버튼 + "HIBI" 타이틀 |
| 이메일 입력 | TextField | hint: "이메일", 클리어 버튼 |
| 비밀번호 입력 | TextField | hint: "비밀번호", obscure 토글, 클리어 버튼 |
| 로그인 버튼 | ElevatedButton | 이메일+비밀번호 모두 입력 시 활성화 |
| 구분선 | Divider + Text | "또는" |
| 소셜 로그인 버튼 | IconButton x3 | 카카오/구글/네이버 (OB-01과 동일 스타일) |
| 회원가입 링크 | TextButton | "계정이 없으신가요? 회원가입" |

#### 상태별 UI
| 상태 | UI |
|------|-----|
| 기본 | 빈 폼, 로그인 버튼 비활성화 |
| 입력 중 | 이메일+비밀번호 모두 입력 시 로그인 버튼 활성화 |
| 로그인 진행 중 | 로그인 버튼에 CircularProgressIndicator |
| 로그인 실패 | SnackBar 또는 TextField 하단 에러 메시지 ("이메일 또는 비밀번호가 올바르지 않습니다") |
| 로그인 성공 | 메인 화면으로 자동 이동 |

#### 인터랙션
- "로그인" 버튼 탭 → API 호출 → 성공 시 메인 이동, 실패 시 에러 표시
- 소셜 로그인 아이콘 탭 → OB-03으로 전환
- "회원가입" 탭 → 기존 회원가입 플로우로 이동
- 뒤로가기 → OB-01 온보딩으로 복귀

### 3. OB-03: 소셜 인증 대기 화면

#### 기본 정보
- Route: 별도 라우트 없음 (오버레이 또는 전체 화면 모달)
- 타입: 대기 화면 (풀스크린)
- 신규 생성

#### UI 레이아웃
```
┌─────────────────────────┐
│                         │
│                         │
│                         │
│    [소셜 서비스 로고]      │  ← 카카오/구글/네이버 로고
│                         │
│   {서비스명}으로          │
│   로그인 중입니다...       │  ← 안내 텍스트
│                         │
│    ◌ (로딩 인디케이터)     │  ← CircularProgressIndicator
│                         │
│                         │
│                         │
│       [취소]             │  ← TextButton
│                         │
└─────────────────────────┘
```

#### UI 요소 상세
| 요소 | 타입 | 설명 |
|------|------|------|
| 소셜 서비스 로고 | Image | 선택한 소셜 서비스 로고 (카카오/구글/네이버) |
| 안내 텍스트 | Text | "{서비스명}으로 로그인 중입니다..." |
| 로딩 인디케이터 | CircularProgressIndicator | 인증 진행 중 표시 |
| 취소 버튼 | TextButton | 소셜 인증 취소 → 이전 화면 복귀 |

#### 상태별 UI
| 상태 | UI |
|------|-----|
| 인증 진행 중 | 로딩 인디케이터 + 안내 텍스트 |
| 인증 성공 (기존 회원) | 짧은 성공 메시지 후 메인 화면 이동 |
| 인증 성공 (신규 회원) | 닉네임 설정 화면으로 이동 (기존 nickname_view.dart 재활용) |
| 인증 실패 | 오류 메시지 SnackBar + 이전 화면 복귀 |
| 사용자 취소 | 이전 화면 복귀 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" 메시지 + 재시도 버튼 |

#### 인터랙션
- 소셜 SDK WebView/앱 전환 → 인증 완료 콜백 수신
- 인증 성공 → JWT 토큰 저장 → 메인 화면 이동 (또는 닉네임 설정)
- "취소" 탭 → 소셜 인증 중단, 이전 화면 복귀
