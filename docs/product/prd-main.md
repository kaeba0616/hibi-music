# hibi PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 제품 비전
매일 한 곡의 JPOP을 추천받고, 팬들과 소통하는 일본 음악 공유 플랫폼

### 1.2 목표 사용자
- JPOP 팬
- 일본 문화 애호가
- 음악 커뮤니티를 원하는 사용자

---

## 2. Feature 목록

### Phase 1: MVP Core

#### F1: Auth & User (인증 및 유저 관리)
- 회원가입/로그인
- 프로필 관리
- JWT 기반 인증

#### F2: Daily Song (오늘의 노래)
- 매일 추천곡 표시
- 좋아요 기능
- 곡 정보 표시

#### F3: Artist (아티스트)
- 아티스트 목록
- 아티스트 상세
- 아티스트 팔로우

#### F4: Calendar (캘린더)
- 월별 추천곡 히스토리
- 날짜별 곡 조회
- 좋아요 필터

---

## 3. Feature 상세 스펙

### F1: Auth & User

#### Acceptance Criteria

##### AC-F1-1: 회원가입
- Given: 비로그인 사용자
- When: 이메일, 비밀번호, 닉네임 입력 후 가입 버튼 클릭
- Then: 계정이 생성되고 로그인 상태가 됨

##### AC-F1-2: 로그인
- Given: 가입된 사용자
- When: 이메일, 비밀번호 입력 후 로그인 버튼 클릭
- Then: JWT 토큰 발급 및 메인 화면으로 이동

##### AC-F1-3: 프로필 조회
- Given: 로그인된 사용자
- When: 프로필 화면 진입
- Then: 닉네임, 프로필 이미지, 가입일 표시

##### AC-F1-4: 프로필 수정
- Given: 로그인된 사용자
- When: 닉네임 또는 프로필 이미지 변경 후 저장
- Then: 변경사항이 저장됨

##### AC-F1-5: 온보딩
- Given: 앱 최초 진입 또는 로그아웃 상태
- When: 앱 시작
- Then: 온보딩 페이지 표시 (메인 이동, 로그인, 회원가입 버튼)

##### AC-F1-6: 소셜 로그인
- Given: 온보딩 또는 로그인 화면
- When: 카카오/구글/네이버 소셜 로그인 버튼 클릭
- Then: 해당 소셜 인증 진행 후 로그인 완료, 메인 화면 이동

##### AC-F1-7: 이메일 인증 (회원가입)
- Given: 회원가입 이메일 입력 화면
- When: 이메일 입력 후 인증 요청
- Then: 이메일 중복 체크 후 인증번호 발송, 인증확인 페이지로 이동

##### AC-F1-8: 인증번호 확인
- Given: 인증확인 페이지
- When: 인증번호 입력 후 확인
- Then: 인증 완료 시 개인정보 등록 페이지로 이동 (이메일 자동 삽입)

##### AC-F1-9: 닉네임 변경 제한
- Given: 로그인된 사용자, 프로필 수정 화면
- When: 닉네임 변경 시도
- Then: 1달 1회 제한 적용, 중복 및 금칙어 불가

##### AC-F1-10: 비밀번호 규칙
- Given: 회원가입 또는 비밀번호 변경 화면
- When: 비밀번호 입력
- Then: 최소 8글자, 영문+숫자+특수기호 모두 포함 필수

##### AC-F1-11: 로그아웃
- Given: 로그인된 사용자
- When: 마이페이지에서 로그아웃 버튼 클릭
- Then: 토큰/세션 삭제, 로그아웃 완료 안내 후 온보딩 페이지로 이동

### F2: Daily Song

#### Acceptance Criteria

##### AC-F2-1: 오늘의 노래 표시
- Given: 로그인된 사용자
- When: 메인 화면 진입
- Then: 오늘 날짜의 추천곡이 표시됨 (곡명, 아티스트, 앨범 이미지)

##### AC-F2-2: 노래 좋아요
- Given: 오늘의 노래가 표시된 상태
- When: 좋아요 버튼 클릭
- Then: 좋아요 상태가 토글되고 UI에 반영됨

##### AC-F2-3: 노래 상세 정보
- Given: 오늘의 노래가 표시된 상태
- When: 노래 카드 클릭
- Then: 상세 화면으로 이동 (가사, 발매일, 앨범 정보 등)

##### AC-F2-4: 과거 노래 없음
- Given: 아직 추천곡이 없는 날짜
- When: 해당 날짜 조회
- Then: "아직 추천곡이 없습니다" 메시지 표시

##### AC-F2-5: 유튜브 영상
- Given: 오늘의 노래 상세 화면
- When: 화면 진입
- Then: 해당 노래의 유튜브 영상이 임베드되어 표시됨

##### AC-F2-6: 연관곡 표시
- Given: 오늘의 노래 상세 화면 하단
- When: 연관곡 영역 확인
- Then: 연관곡 목록이 표시됨 (제목, 아티스트, 선정 이유), 클릭 시 해당 곡 소개 페이지로 이동

##### AC-F2-7: 좋아요 곡 모아보기
- Given: 로그인된 사용자
- When: 좋아요 곡 목록 화면 진입
- Then: 좋아요 누른 오늘의 곡 목록이 표시됨 (제목, 아티스트)

- Given: 아티스트 목록 화면

### F4: Calendar

#### Acceptance Criteria

##### AC-F4-1: 월별 캘린더 표시
- Given: 로그인된 사용자
- When: 캘린더 탭 진입
- Then: 현재 월의 캘린더가 표시되고, 추천곡이 있는 날짜에 마커 표시

- Given: 캘린더가 표시된 상태
- When: 특정 날짜 클릭
- Then: 해당 날짜의 추천곡이 하단에 표시됨

##### AC-F4-3: 월 변경
- Given: 캘린더가 표시된 상태
- When: 이전/다음 월 버튼 클릭
- Then: 해당 월의 캘린더로 이동

##### AC-F4-4: 좋아요 필터
- Given: 캘린더가 표시된 상태
- When: "좋아요만" 필터 적용
- Then: 좋아요 표시한 날짜만 마커 표시

## 3.5 Phase 2 Feature 상세 스펙

### F5: Post

> **Note**: 이 기능은 Notion PRD에 아직 미반영 상태입니다. 추후 Notion PRD에 추가 예정입니다.

#### Acceptance Criteria

##### AC-F5-1: 게시글 목록 조회
- Given: 로그인된 사용자
- When: 피드(커뮤니티) 탭 진입
- Then: 최신 게시글 목록이 표시됨 (작성자, 내용, 작성시간, 좋아요/댓글 수)

##### AC-F5-2: 게시글 작성
- Given: 로그인된 사용자
- When: 게시글 작성 버튼 클릭 후 내용 입력하고 등록
- Then: 게시글이 생성되고 목록에 표시됨

##### AC-F5-3: 게시글 상세 조회
- Given: 게시글 목록이 표시된 상태
- When: 게시글 클릭
- Then: 게시글 상세 화면으로 이동 (전체 내용, 이미지, 댓글 목록)

##### AC-F5-4: 노래 태그
- Given: 게시글 작성 화면
- When: 노래 태그 버튼 클릭 후 노래 선택
- Then: 게시글에 노래 정보가 첨부됨 (제목, 아티스트, 앨범 이미지)

##### AC-F5-5: 게시글 수정
- Given: 본인이 작성한 게시글 상세 화면
- When: 수정 버튼 클릭 후 내용 변경하고 저장
- Then: 게시글이 수정됨

##### AC-F5-6: 게시글 삭제
- Given: 본인이 작성한 게시글 상세 화면
- When: 삭제 버튼 클릭 후 확인
- Then: 게시글이 삭제되고 목록으로 이동

##### AC-F5-7: 게시글 좋아요
- Given: 게시글 목록 또는 상세 화면
- When: 좋아요 버튼 클릭
- Then: 좋아요 상태가 토글되고 UI에 반영됨

### F6: Comment & Reaction

#### Acceptance Criteria

##### AC-F6-1: 댓글 목록 조회
- Given: 게시글 상세 화면
- When: 화면 진입
- Then: 해당 게시글의 댓글 목록이 표시됨 (작성자, 내용, 작성시간)

##### AC-F6-2: 댓글 작성
- Given: 게시글 상세 화면
- When: 댓글 입력 후 전송 버튼 클릭
- Then: 댓글이 생성되고 목록에 표시됨

##### AC-F6-3: 댓글 삭제
- Given: 본인이 작성한 댓글
- When: 삭제 버튼 클릭 후 확인
- Then: 댓글이 삭제되고 목록에서 제거됨

##### AC-F6-4: 대댓글 작성
- Given: 게시글 상세 화면의 댓글
- When: 답글 버튼 클릭 후 내용 입력하고 전송
- Then: 대댓글이 해당 댓글 아래에 표시됨

##### AC-F6-5: 댓글 좋아요
- Given: 댓글 목록
- When: 댓글의 좋아요 버튼 클릭
- Then: 좋아요 상태가 토글되고 UI에 반영됨

##### AC-F6-6: 댓글 추천 Top3
- Given: 댓글 목록 화면
- When: 화면 진입
- Then: 추천이 가장 많은 댓글 3개가 최상단에 노출됨

##### AC-F6-7: 댓글 신고
- Given: 로그인된 사용자
- When: 댓글의 신고 버튼 클릭
- Then: 신고 접수 처리됨

##### AC-F6-8: 부적절 댓글 필터링
- Given: 댓글 목록
- When: 부적절한 내용이 포함된 댓글
- Then: 필터링 적용되어 노출되지 않음

### F7: Follow

> **Note**: 이 기능은 Notion PRD에 아직 미반영 상태입니다. 추후 Notion PRD에 추가 예정입니다.

#### Acceptance Criteria

##### AC-F7-1: 사용자 프로필 조회
- Given: 로그인된 사용자
- When: 다른 사용자의 프로필 화면 진입
- Then: 프로필 정보(닉네임, 프로필 이미지, 게시글 수, 팔로워/팔로잉 수)가 표시됨

##### AC-F7-2: 사용자 팔로우
- Given: 다른 사용자의 프로필 화면
- When: 팔로우 버튼 클릭
- Then: 팔로우 상태가 되고 UI에 반영됨

##### AC-F7-3: 사용자 언팔로우
- Given: 팔로우 중인 사용자의 프로필 화면
- When: 팔로잉 버튼 클릭 후 확인
- Then: 팔로우가 해제되고 UI에 반영됨

##### AC-F7-4: 팔로워 목록 조회
- Given: 프로필 화면
- When: 팔로워 수 탭
- Then: 팔로워 목록이 표시됨 (닉네임, 프로필 이미지, 팔로우 상태)

##### AC-F7-5: 팔로잉 목록 조회
- Given: 프로필 화면
- When: 팔로잉 수 탭
- Then: 팔로잉 목록이 표시됨 (닉네임, 프로필 이미지, 팔로우 상태)

##### AC-F7-6: 팔로우 피드
- Given: 로그인된 사용자
- When: 피드에서 "팔로잉" 필터 선택
- Then: 팔로우한 사용자의 게시글만 표시됨

##### AC-F7-7: 사용자 게시글 목록
- Given: 사용자 프로필 화면
- When: 화면 스크롤
- Then: 해당 사용자가 작성한 게시글 목록이 표시됨

### F8: Search

#### Acceptance Criteria

##### AC-F8-1: 통합 검색
- Given: 로그인된 사용자
- When: 검색창에 키워드 입력 후 검색
- Then: 노래, 아티스트, 게시글, 사용자 결과가 카테고리별로 표시됨

##### AC-F8-2: 카테고리별 필터
- Given: 검색 결과가 표시된 상태
- When: 특정 카테고리 탭 선택 (노래/아티스트/게시글/사용자)
- Then: 해당 카테고리의 결과만 표시됨

##### AC-F8-3: 최근 검색어
- Given: 검색 화면 진입
- When: 검색창에 포커스
- Then: 최근 검색어 목록이 표시됨 (최대 10개)

##### AC-F8-4: 검색 결과 없음
- Given: 검색 실행
- When: 검색 결과가 없는 경우
- Then: "검색 결과가 없습니다" 메시지와 다른 키워드 제안

##### AC-F8-5: 검색 결과 상세 이동
- Given: 검색 결과가 표시된 상태
- When: 결과 항목 클릭
- Then: 해당 항목의 상세 화면으로 이동

## 3.6 Phase 3 Feature 상세 스펙

### F9: FAQ

#### Acceptance Criteria

##### AC-F9-1: FAQ 목록 조회
- Given: 로그인/비로그인 사용자
- When: 설정 > FAQ 메뉴 진입
- Then: FAQ 목록이 카테고리별로 그룹화되어 표시됨

##### AC-F9-2: FAQ 카테고리 필터
- Given: FAQ 목록 화면
- When: 특정 카테고리 탭 선택
- Then: 해당 카테고리의 FAQ만 표시됨

##### AC-F9-3: FAQ 상세 보기
- Given: FAQ 목록이 표시된 상태
- When: FAQ 항목 클릭
- Then: 질문과 답변이 확장되어 표시됨 (아코디언 방식)

##### AC-F9-4: FAQ 검색
- Given: FAQ 화면
- When: 검색창에 키워드 입력
- Then: 질문/답변에 키워드가 포함된 FAQ가 필터링되어 표시됨

##### AC-F9-5: FAQ 없음 상태
- Given: FAQ 화면
- When: 검색 결과가 없거나 등록된 FAQ가 없는 경우
- Then: "검색 결과가 없습니다" 또는 "등록된 FAQ가 없습니다" 메시지 표시

### F10: Question (문의하기)

#### Acceptance Criteria

##### AC-F10-1: 문의하기 화면 진입
- Given: 로그인된 사용자
- When: FAQ 화면의 "문의하기" 버튼 또는 설정 > 문의하기 메뉴 클릭
- Then: 문의하기 작성 화면이 표시됨

##### AC-F10-2: 문의 유형 선택
- Given: 문의하기 작성 화면
- When: 문의 유형 선택 영역 탭
- Then: 문의 유형 목록이 표시됨 (계정, 서비스 이용, 버그 신고, 기능 제안, 기타)

##### AC-F10-3: 문의 작성
- Given: 문의하기 작성 화면
- When: 유형 선택, 제목, 내용 입력 후 제출 버튼 클릭
- Then: 문의가 등록되고 확인 메시지 표시

##### AC-F10-4: 문의 내역 조회
- Given: 로그인된 사용자
- When: 설정 > 문의 내역 메뉴 진입
- Then: 본인이 작성한 문의 목록이 표시됨 (유형, 제목, 작성일, 답변 상태)

##### AC-F10-5: 문의 상세 조회
- Given: 문의 내역 화면
- When: 문의 항목 클릭
- Then: 문의 상세 내용과 답변(있는 경우)이 표시됨

##### AC-F10-6: 입력 유효성 검사
- Given: 문의하기 작성 화면
- When: 필수 항목(유형, 제목, 내용) 미입력 시 제출 시도
- Then: 에러 메시지와 해당 필드 강조 표시

##### AC-F10-7: 비로그인 사용자 안내
- Given: 비로그인 사용자
- When: 문의하기 접근 시도
- Then: 로그인 필요 안내와 로그인 화면으로 이동 옵션 표시

##### AC-F10-8: 문의 작성 제한
- Given: 로그인된 사용자
- When: 문의 작성 시
- Then: 내용 최소 10자 이상, 일일 작성 가능 문의 수 3개 제한

### F11: Report (신고)

> **Note**: 이 기능은 Notion PRD에 아직 미반영 상태입니다. 추후 Notion PRD에 추가 예정입니다.

#### Acceptance Criteria

##### AC-F11-1: 게시글 신고
- Given: 로그인된 사용자
- When: 게시글의 더보기 메뉴에서 "신고하기" 선택
- Then: 신고 사유 선택 화면이 표시됨

##### AC-F11-2: 댓글 신고
- Given: 로그인된 사용자
- When: 댓글의 더보기 메뉴에서 "신고하기" 선택
- Then: 신고 사유 선택 화면이 표시됨

##### AC-F11-3: 사용자 신고
- Given: 로그인된 사용자
- When: 사용자 프로필의 더보기 메뉴에서 "신고하기" 선택
- Then: 신고 사유 선택 화면이 표시됨

##### AC-F11-4: 신고 사유 선택
- Given: 신고 화면
- When: 신고 사유 선택 (스팸/광고, 욕설/비방, 불쾌한 내용, 저작권 침해, 기타)
- Then: 선택한 사유가 표시되고 제출 버튼 활성화

##### AC-F11-5: 신고 상세 내용 입력
- Given: "기타" 사유 선택
- When: 상세 내용 입력
- Then: 입력 내용이 저장됨 (선택 사항)

##### AC-F11-6: 신고 제출
- Given: 신고 사유가 선택된 상태
- When: 제출 버튼 클릭
- Then: 신고가 접수되고 확인 메시지 표시

##### AC-F11-7: 중복 신고 방지
- Given: 이미 신고한 콘텐츠/사용자
- When: 다시 신고 시도
- Then: "이미 신고한 항목입니다" 메시지 표시

##### AC-F11-8: 본인 콘텐츠 신고 방지
- Given: 로그인된 사용자
- When: 본인 게시글/댓글에서 신고 메뉴 확인
- Then: 신고하기 메뉴가 표시되지 않음

### F12: Management (관리자 기능)

#### Acceptance Criteria

##### AC-F12-1: 관리자 대시보드 접근
- Given: ADMIN 권한을 가진 사용자
- When: 관리자 메뉴에 접근
- Then: 관리자 대시보드가 표시됨 (통계 요약)

##### AC-F12-2: 신고 목록 조회
- Given: 관리자 대시보드
- When: 신고 관리 메뉴 선택
- Then: 대기중인 신고 목록이 표시됨 (최신순)

##### AC-F12-3: 신고 상세 조회
- Given: 신고 목록 화면
- When: 특정 신고 항목 클릭
- Then: 신고 상세 정보와 신고 대상 콘텐츠가 표시됨

##### AC-F12-4: 신고 처리 (조치)
- Given: 신고 상세 화면
- When: 조치 버튼 클릭 (삭제, 경고, 기각)
- Then: 신고 상태가 변경되고 필요시 콘텐츠가 삭제됨

##### AC-F12-5: 문의 목록 조회
- Given: 관리자 대시보드
- When: 문의 관리 메뉴 선택
- Then: 미답변 문의 목록이 표시됨 (접수순)

##### AC-F12-6: 문의 답변 작성
- Given: 문의 상세 화면
- When: 답변 작성 후 제출
- Then: 답변이 저장되고 문의 상태가 '답변완료'로 변경됨

##### AC-F12-7: FAQ 목록 관리
- Given: 관리자 대시보드
- When: FAQ 관리 메뉴 선택
- Then: FAQ 목록이 표시됨 (카테고리별)

##### AC-F12-8: FAQ 추가/수정/삭제
- Given: FAQ 관리 화면
- When: 추가/수정/삭제 버튼 클릭
- Then: FAQ 항목이 추가/수정/삭제됨

##### AC-F12-9: 사용자 목록 조회
- Given: 관리자 대시보드
- When: 사용자 관리 메뉴 선택
- Then: 사용자 목록이 표시됨 (검색/필터 가능)

##### AC-F12-10: 사용자 제재 (정지/탈퇴)
- Given: 사용자 상세 화면
- When: 제재 버튼 클릭
- Then: 사용자 계정이 정지되거나 강제 탈퇴됨

##### AC-F12-11: 일반 사용자 접근 차단
- Given: USER 권한을 가진 사용자
- When: 관리자 URL 직접 접근 시도
- Then: 403 Forbidden 또는 메인 화면으로 리다이렉트

##### AC-F12-12: 오늘의 곡 등록 상세
- Given: 관리자 오늘의 곡 등록 화면
- When: 곡 정보 입력
- Then: 곡 제목(한글/영어/일본어), 아티스트(자동완성), 스토리, 가사, 연관곡 리스트 등록 가능

##### AC-F12-13: 예약 게시
- Given: 관리자 오늘의 곡 관리 화면
- When: 예약 게시 설정
- Then: 공개 시간 설정 가능, 미작성 곡은 예약 불가

##### AC-F12-14: 관리자 댓글 관리
- Given: 관리자 댓글 목록 화면
- When: 댓글 목록 조회
- Then: 작성일, 내용, 작성자 표시, 삭제 가능
